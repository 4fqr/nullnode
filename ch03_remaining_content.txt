<!-- Topic 4: User Management & sudo -->
<div style="background: linear-gradient(135deg, rgba(255,200,0,0.15) 0%, rgba(255,150,0,0.15) 100%); padding: 3rem; border-radius: 20px; margin: 3rem 0; border: 2px solid rgba(255,200,0,0.3);">
    <h2 style="color: #ffc800; font-size: 1.8rem; margin-bottom: 1.5rem;">üë• Topic 4: User Management & sudo ‚Äî Controlling Access</h2>
    
    <h3 style="color: #ffd54f; font-size: 1.4rem; margin: 2rem 0 1rem 0;">User Accounts</h3>
    <p style="font-size: 1.1rem; line-height: 1.8; margin-bottom: 1.5rem;">
        Linux is multi-user by design. Every process runs as a user. Every file belongs to a user. Understanding user management = understanding system security.
    </p>

    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code># Create new user
sudo useradd -m -s /bin/bash alice

# Set password
sudo passwd alice

# Add user to group
sudo usermod -aG sudo alice  # Add to sudoers
sudo usermod -aG docker alice  # Add to docker group

# Delete user
sudo userdel -r alice  # -r removes home directory

# View user info
id alice
whoami
w  # Who's logged in
last  # Login history
</code></pre>
    </div>

    <h3 style="color: #ffd54f; font-size: 1.4rem; margin: 2rem 0 1rem 0;">The /etc/passwd File</h3>
    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code>root:x:0:0:root:/root:/bin/bash
alice:x:1001:1001:Alice Smith:/home/alice:/bin/bash
</code></pre>
    </div>
    <p style="font-size: 1.05rem; line-height: 1.8; margin-bottom: 1.5rem;">
        Format: <code style="background: rgba(0,0,0,0.4); padding: 0.2rem 0.5rem; border-radius: 5px;">username:password:UID:GID:comment:home:shell</code>
    </p>
    <ul style="font-size: 1.05rem; line-height: 1.9; margin: 0 0 1.5rem 2rem;">
        <li><strong>password:</strong> <code>x</code> means password stored in <code>/etc/shadow</code></li>
        <li><strong>UID 0:</strong> Root user (superuser)</li>
        <li><strong>UID 1-999:</strong> System accounts</li>
        <li><strong>UID 1000+:</strong> Regular users</li>
    </ul>

    <h3 style="color: #ffd54f; font-size: 1.4rem; margin: 2rem 0 1rem 0;">The /etc/shadow File</h3>
    <p style="font-size: 1.05rem; line-height: 1.8; margin-bottom: 1.5rem;">
        Contains actual password hashes. Only readable by root.
    </p>
    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code>alice:$6$randomsalt$hashedpassword:19000:0:99999:7:::
</code></pre>
    </div>

    <h3 style="color: #ffd54f; font-size: 1.4rem; margin: 2rem 0 1rem 0;">sudo ‚Äî Superuser Do</h3>
    <p style="font-size: 1.1rem; line-height: 1.8; margin-bottom: 1.5rem;">
        <code style="background: rgba(0,0,0,0.4); padding: 0.2rem 0.5rem; border-radius: 5px; color: #00ccff;">sudo</code> lets regular users execute commands as root (temporarily).
    </p>
    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code># Run command as root
sudo apt update

# Open root shell
sudo -i  # Login shell
sudo -s  # Non-login shell

# Run as different user
sudo -u alice command

# Edit sudoers file (SAFELY)
sudo visudo  # Never edit /etc/sudoers directly!

# Check sudo privileges
sudo -l
</code></pre>
    </div>

    <h3 style="color: #ffd54f; font-size: 1.4rem; margin: 2rem 0 1rem 0;">sudoers Configuration</h3>
    <div style="background: rgba(0,0,0,0.3); padding: 2rem; border-radius: 15px; margin: 1.5rem 0;">
        <p style="font-size: 1.05rem; line-height: 1.8; margin-bottom: 1rem;">
            <strong>Example /etc/sudoers entries:</strong>
        </p>
        <pre style="margin: 0; color: #00ccff; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code># Allow alice to run any command
alice ALL=(ALL:ALL) ALL

# Allow developers group to run commands without password
%developers ALL=(ALL) NOPASSWD: ALL

# Allow specific commands only
bob ALL=(ALL) /usr/bin/apt, /usr/bin/systemctl
</code></pre>
    </div>

    <div style="background: rgba(255,200,0,0.15); padding: 1.5rem; border-radius: 10px; margin: 2rem 0; border-left: 4px solid #ffc800;">
        <h4 style="color: #ffc800; margin-bottom: 0.5rem;">üî• Why This Matters for Hacking</h4>
        <p style="font-size: 1.05rem; line-height: 1.8; margin: 0;">
            Privilege escalation often starts with misconfigured sudo. <code style="background: rgba(0,0,0,0.4); padding: 0.2rem 0.5rem; border-radius: 5px;">sudo -l</code> reveals what you can run as root. Find <code>NOPASSWD</code> entries? Gold mine. Exploitable binaries in sudoers? Root access. Always check sudo configs during penetration tests.
        </p>
    </div>

<!-- Topic 5: Process Management -->
<div style="background: linear-gradient(135deg, rgba(0,255,136,0.15) 0%, rgba(0,200,100,0.15) 100%); padding: 3rem; border-radius: 20px; margin: 3rem 0; border: 2px solid rgba(0,255,136,0.3);">
    <h2 style="color: #00ff88; font-size: 1.8rem; margin-bottom: 1.5rem;">‚öôÔ∏è Topic 5: Process Management ‚Äî Controlling Running Programs</h2>
    
    <h3 style="color: #33ffa5; font-size: 1.4rem; margin: 2rem 0 1rem 0;">What IS a Process?</h3>
    <p style="font-size: 1.1rem; line-height: 1.8; margin-bottom: 1.5rem;">
        A process is a <strong>running instance of a program</strong>. When you execute <code style="background: rgba(0,0,0,0.4); padding: 0.2rem 0.5rem; border-radius: 5px;">ls</code>, the kernel creates a process to run it.
    </p>
    <p style="font-size: 1.1rem; line-height: 1.8; margin-bottom: 1.5rem;">
        Every process has a <strong>PID</strong> (Process ID) ‚Äî a unique number identifying it.
    </p>

    <h3 style="color: #33ffa5; font-size: 1.4rem; margin: 2rem 0 1rem 0;">Viewing Processes</h3>
    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code># List all processes
ps aux  # BSD style
ps -ef  # System V style

# Interactive process viewer
top
htop  # More user-friendly (install with apt/yum)

# Process tree (parent-child relationships)
pstree

# Find specific process
ps aux | grep apache2
pgrep apache2  # Returns PIDs only
</code></pre>
    </div>

    <h3 style="color: #33ffa5; font-size: 1.4rem; margin: 2rem 0 1rem 0;">Killing Processes</h3>
    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code># Kill by PID
kill 1234

# Force kill (SIGKILL)
kill -9 1234

# Kill by name
killall firefox
pkill firefox

# Common signals
kill -15 1234  # SIGTERM (graceful shutdown)
kill -9 1234   # SIGKILL (immediate termination)
kill -1 1234   # SIGHUP (reload configuration)
</code></pre>
    </div>

    <h3 style="color: #33ffa5; font-size: 1.4rem; margin: 2rem 0 1rem 0;">Background & Foreground Jobs</h3>
    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code># Run in background
command &

# Suspend current process (Ctrl+Z), then background it
bg

# Bring to foreground
fg

# List jobs
jobs

# Detach process from terminal (keeps running after logout)
nohup command &
</code></pre>
    </div>

    <h3 style="color: #33ffa5; font-size: 1.4rem; margin: 2rem 0 1rem 0;">systemd & Services</h3>
    <p style="font-size: 1.1rem; line-height: 1.8; margin-bottom: 1.5rem;">
        <code style="background: rgba(0,0,0,0.4); padding: 0.2rem 0.5rem; border-radius: 5px; color: #00ccff;">systemd</code> is the modern init system managing services (daemons) on most Linux distributions.
    </p>
    <div style="background: rgba(0,0,0,0.4); padding: 2rem; border-radius: 10px; margin: 1.5rem 0;">
        <pre style="margin: 0; color: #00ff88; font-family: 'Courier New', monospace; font-size: 1rem; line-height: 1.8;"><code># Start/stop services
sudo systemctl start apache2
sudo systemctl stop apache2
sudo systemctl restart apache2
sudo systemctl reload apache2  # Reload config without restart

# Enable/disable autostart on boot
sudo systemctl enable apache2
sudo systemctl disable apache2

# Check service status
systemctl status apache2

# List all services
systemctl list-units --type=service

# View service logs
journalctl -u apache2 -f  # Follow logs live
</code></pre>
    </div>

    <div style="background: rgba(255,200,0,0.15); padding: 1.5rem; border-radius: 10px; margin: 2rem 0; border-left: 4px solid #ffc800;">
        <h4 style="color: #ffc800; margin-bottom: 0.5rem;">üî• Why This Matters for Hacking</h4>
        <p style="font-size: 1.05rem; line-height: 1.8; margin: 0;">
            After exploiting a system, you need to understand what's running. <code style="background: rgba(0,0,0,0.4); padding: 0.2rem 0.5rem; border-radius: 5px;">ps aux</code> reveals services you can target. Want persistence? Hijack a systemd service. Need to hide? Kill logging daemons. Process management is attack surface reconnaissance.
        </p>
    </div>
</div>
