Chapter 03: Linux Mastery

Complete Linux Expertise - From Distros to System Hardening

Welcome to the Linux Universe

Linux isn't just an operating system - it's the backbone of the entire cybersecurity world. From hacking labs to production servers, from bug bounty targets to security research platforms, Linux is everywhere.

Think of Linux mastery like learning to navigate a vast city where every street leads to a different capability. At first, the terminal feels intimidating - just a blinking cursor waiting for input. But soon, you'll realize this "cursor" is your direct hotline to the machine's brain.

In this chapter, we'll transform you from a Linux tourist into a Linux native. You'll understand the file system, manage permissions, and automate tasks like a pro.

Topic 1: Linux Distributions

Imagine Linux as a car engine (the kernel). A distribution is the entire car - complete with seats, dashboard, and paint job. Different manufacturers take the same engine and build completely different vehicles.

The Main Families

Debian Family (apt)

Ubuntu: Beginner-friendly, massive community, great hardware support.

Debian: More stable, minimal, server-focused.

Kali Linux: Pre-loaded with 600+ security tools for penetration testing.

Red Hat Family (dnf/yum)

Fedora: Bleeding-edge features for developers.
RHEL: Enterprise-grade, ultra-stable.
CentOS: Free RHEL alternative.

Arch Family (pacman)

Arch Linux: DIY builder's dream, rolling release.
Manjaro: Arch made easy.
BlackArch: 2,800+ security tools.

Topic 2: File System Hierarchy

Everything Starts at Root (/)

Unlike Windows with multiple drives, Linux has only one root: /

Key Directories

  • /bin - Essential commands (ls, cat, cp)
  • /etc - System configuration files
  • /home - User directories
  • /var - Variable data (logs, web content)
  • /tmp - Temporary files (cleared on reboot)
  • /usr - User programs and utilities
  • /root - Root user's home

Basic Navigation

pwd                    # Print working directory
ls -lah                # List files with details
cd /etc                # Change directory
cd ~                   # Go to home
cd -                   # Go to previous
find / -name "*.conf"  # Find files
grep -r "text" /etc    # Search in files

Topic 3: Permissions

Every file has three permission sets: User, Group, and Others.

Permission Types

  • r (4) - Read
  • w (2) - Write
  • x (1) - Execute
chmod 755 script.sh        # rwxr-xr-x
chmod 644 file.txt         # rw-r--r--
chmod u+x script.sh        # Add execute for user
chown alice:dev file.txt   # Change owner and group

# Find SUID binaries (privilege escalation)
find / -perm -4000 2>/dev/null

Topic 4: User Management & sudo

sudo useradd -m alice       # Create user
sudo passwd alice           # Set password
sudo usermod -aG sudo alice # Add to sudo group
sudo userdel -r alice       # Delete user

id                          # View current user
whoami                      # Display username
sudo -l                     # Check sudo privileges

Key files: /etc/passwd (user info), /etc/shadow (password hashes)

Topic 5: Process Management

ps aux                  # List all processes
top                     # Interactive viewer
htop                    # Better top

kill 1234               # Terminate process
kill -9 1234            # Force kill
killall firefox         # Kill by name

# Services (systemd)
systemctl start apache2
systemctl stop apache2
systemctl status apache2
systemctl enable apache2  # Autostart on boot

Topic 6: Package Management

Debian/Ubuntu (apt)

sudo apt update
sudo apt upgrade
sudo apt install nmap
sudo apt remove nmap
apt search network

Red Hat/CentOS (dnf)

sudo dnf install nmap
sudo dnf update
sudo dnf remove nmap

Arch (pacman)

sudo pacman -S nmap
sudo pacman -Syu
sudo pacman -R nmap

Topic 7: Bash Scripting

Basic Script Structure

#!/bin/bash
# My first script

name="Alice"
echo "Hello, $name"

# Loops
for i in {1..5}; do
    echo "Number $i"
done

# Conditionals
if [ -f "/etc/passwd" ]; then
    echo "File exists"
fi

# Make executable
chmod +x script.sh
./script.sh

Topic 8: Networking & SSH

# Network commands
ip addr show
netstat -tuln
ss -tuln
ping google.com
traceroute google.com

# SSH
ssh user@server
ssh -p 2222 user@server
scp file.txt user@server:/path
ssh-keygen -t rsa -b 4096
ssh-copy-id user@server

# Firewall
sudo ufw status
sudo ufw allow 22/tcp
sudo ufw enable
