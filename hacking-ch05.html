Chapter 05

Networking Deep Dive

Master the protocols, packets, and infrastructure that power the internet - and learn how hackers exploit every layer.

Networking is the foundation of hacking. Every attack travels over a network. Every vulnerability exists because data moves between systems. You can't hack what you don't understand, and you can't understand hacking without understanding networks.

üì¨ Networks Are Like Postal Systems

IP addresses are like street addresses - they identify destinations.

Ports are like apartment numbers - they identify specific services.

Packets are like letters - they carry data from source to destination.

Protocols are like postal service rules - they define how communication works.

üéØ Why Networking Matters for Hackers

Network knowledge separates script kiddies from real hackers. Understanding TCP/IP, subnetting, routing, and protocols lets you: find hidden systems, bypass firewalls, intercept traffic, perform man-in-the-middle attacks, and exploit protocol weaknesses.

The OSI Model

The OSI (Open Systems Interconnection) model divides networking into 7 layers. Each layer has specific responsibilities and attack vectors.

7. Application Layer

HTTP, FTP, DNS, SMTP - User-facing protocols

Attacks: SQL injection, XSS, command injection, API abuse

6. Presentation Layer

Data formatting, encryption, compression

Attacks: SSL/TLS attacks, encoding exploits

5. Session Layer

Session management between applications

Attacks: Session hijacking, session fixation

4. Transport Layer

TCP, UDP - Reliable data transfer

Attacks: SYN flood, port scanning, TCP hijacking

3. Network Layer

IP, ICMP, routing between networks

Attacks: IP spoofing, routing attacks, ICMP flood

2. Data Link Layer

MAC addressing, switching, frames

Attacks: ARP spoofing, MAC flooding, VLAN hopping

1. Physical Layer

Cables, signals, bits on the wire

Attacks: Wire tapping, jamming, physical access

TCP/IP Protocol Stack

The TCP/IP model is what the internet actually uses - a simpler 4-layer model.

IP Addresses

Every device on a network needs an IP address. Think of it as a phone number for computers.

IPv4: 192.168.1.100 (32-bit, 4 billion addresses) IPv6: 2001:0db8::1 (128-bit, basically infinite) Private IP ranges (not routable on internet): 10.0.0.0 - 10.255.255.255 172.16.0.0 - 172.31.255.255 192.168.0.0 - 192.168.255.255

TCP vs UDP

TCP (Transmission Control Protocol)

Reliable: Guarantees delivery, orders packets, checks errors

Use cases: Web browsing, file transfer, email

Slower but accurate

UDP (User Datagram Protocol)

Unreliable: No delivery guarantee, no ordering, no error checking

Use cases: Streaming, gaming, DNS, VoIP

Faster but may lose data

Three-Way Handshake (TCP Connection)

Client ‚Üí Server: SYN (Synchronize) "Hey, I want to connect. My sequence number is 1000." Server ‚Üí Client: SYN-ACK (Synchronize-Acknowledge) "OK! My sequence number is 2000. I got your 1000." Client ‚Üí Server: ACK (Acknowledge) "Great! I got your 2000. We're connected!" Now data can flow both ways.

‚ö†Ô∏è Hacker Perspective: SYN Flood Attack

Send thousands of SYN packets but never send the final ACK. The server keeps half-open connections in memory, waiting for ACKs that never come. Eventually the server runs out of resources and crashes - denial of service attack.

Essential Network Protocols

DNS (Domain Name System)

Translates domain names to IP addresses. Like a phone book for the internet.

You type: google.com DNS resolves: 172.217.14.206 Your browser connects to that IP

DNS Attacks

DNS Spoofing: Fake DNS responses redirect users to malicious sites

DNS Tunneling: Hide data inside DNS queries to bypass firewalls

DNS Amplification: DDoS attack using open DNS resolvers

ARP (Address Resolution Protocol)

Maps IP addresses to MAC addresses on local networks.

Device: "Who has IP 192.168.1.1?" Router: "I do! My MAC is AA:BB:CC:DD:EE:FF"

ARP Spoofing (Man-in-the-Middle)

Send fake ARP responses claiming you're the router. All traffic flows through you before reaching the real router. You can see and modify everything - passwords, sessions, messages.

ICMP (Internet Control Message Protocol)

Used for network diagnostics and error messages. Powers ping command.

# Ping checks if a host is alive ping 192.168.1.1 # Traceroute shows the path packets take traceroute google.com

ICMP in Hacking

ICMP can reveal network topology, detect firewalls, discover live hosts, and even be used for data exfiltration (ICMP tunneling).

Network Scanning & Reconnaissance

Before you attack, you need to know what's there. Scanning discovers hosts, open ports, services, and vulnerabilities.

Host Discovery

# Ping sweep - find live hosts nmap -sn 192.168.1.0/24 # ARP scan (local network only) arp-scan --localnet # Detect all devices on network netdiscover -r 192.168.1.0/24

Port Scanning with Nmap

Nmap is the industry standard for network scanning.

# Scan common ports nmap 192.168.1.100 # Scan ALL ports (1-65535) nmap -p- 192.168.1.100 # Service version detection nmap -sV 192.168.1.100 # Operating system detection nmap -O 192.168.1.100 # Aggressive scan (OS, version, scripts, traceroute) nmap -A 192.168.1.100 # Stealth SYN scan (doesn't complete handshake) nmap -sS 192.168.1.100 # UDP scan nmap -sU 192.168.1.100

Scan Types Explained

TCP Connect Scan

Completes full TCP handshake. Loud and obvious but works through any firewall.

nmap -sT target

SYN Scan (Stealth)

Sends SYN, waits for SYN-ACK, then sends RST instead of ACK. Stealthier - doesn't log on many systems.

nmap -sS target

NULL/FIN/XMAS Scans

Send malformed packets. Open ports ignore them, closed ports send RST. Bypasses some firewalls.

nmap -sN target

Idle Scan

Uses a zombie host to scan target. Ultimate stealth - target logs show zombie's IP, not yours.

nmap -sI zombie target

‚ö†Ô∏è Legal Warning

Only scan networks you own or have explicit written permission to test. Unauthorized network scanning is illegal in most countries and can result in criminal charges. Companies actively monitor for scans and will prosecute.

Common Network Attacks

Man-in-the-Middle (MITM)

Attacker positions themselves between two communicating parties, intercepting and potentially modifying all traffic.

# ARP spoofing with arpspoof (educational purposes only) # Tell victim you're the router arpspoof -i eth0 -t 192.168.1.50 192.168.1.1 # Tell router you're the victim arpspoof -i eth0 -t 192.168.1.1 192.168.1.50 # Enable packet forwarding so traffic still works echo 1 > /proc/sys/net/ipv4/ip_forward # Now all traffic flows through you!

Denial of Service (DoS)

Overwhelm a target system with traffic until it can't respond to legitimate requests.

SYN Flood

Send massive amounts of SYN packets, never complete handshake. Target's connection table fills up.

UDP Flood

Spam UDP packets to random ports. Target wastes resources checking each one.

ICMP Flood

Send massive ping requests. Old but sometimes effective.

DDoS (Distributed)

Same attacks but from thousands of infected computers (botnet). Nearly impossible to stop.

Packet Sniffing

Capture and analyze network traffic to extract sensitive information.

# Wireshark - GUI packet analyzer (best for learning) wireshark # tcpdump - command-line packet capture tcpdump -i eth0 -n # Capture HTTP traffic tcpdump -i eth0 'tcp port 80' # Capture and save to file tcpdump -i eth0 -w capture.pcap # Read captured file tcpdump -r capture.pcap

What Can You See in Packet Captures?

  • Unencrypted passwords (HTTP, FTP, Telnet)
  • Email content and attachments
  • Session cookies
  • Visited websites
  • File transfers
  • DNS queries (what sites users visit)

This is why HTTPS is critical. Encrypted traffic shows that communication happened, but not what was said.

Session Hijacking

Steal an active session by capturing session tokens/cookies.

# Capture traffic to find session cookies tcpdump -i eth0 -A | grep "Cookie:" # Use stolen cookie to impersonate user # Copy cookie value into your browser # You're now logged in as the victim

DNS Hijacking

Redirect DNS queries to malicious servers.

# Router-level: Change router's DNS settings to malicious DNS server # All devices on network now use attacker's DNS # Attacker can redirect any domain to any IP # Local-level: Modify /etc/hosts or C:\Windows\System32\drivers\etc\hosts 192.168.1.100 bank.com # Victim types bank.com, goes to attacker's fake site

Defense & Detection

üõ°Ô∏è Prevent MITM

  • Use HTTPS everywhere
  • Static ARP entries
  • Network segmentation
  • Encrypted VPN tunnels

üîç Detect Attacks

  • IDS/IPS systems (Snort, Suricata)
  • Monitor ARP tables for changes
  • Watch for port scans
  • Log all network activity

üîí Secure Services

  • Close unused ports
  • Strong firewall rules
  • Disable unnecessary services
  • Keep software updated

üìä Monitor Everything

  • Baseline normal traffic
  • Alert on anomalies
  • Regular network audits
  • Honeypots to detect attackers
