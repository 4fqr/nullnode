Chapter 03: Control Flow & Functions

Making Decisions and Creating Reusable Code Recipes

Part 1: What IS Control Flow?

So far, your Python programs have been like reading a book - they execute line by line, from top to bottom, no questions asked: 

name = "Alice"     # Line 1 runs first
age = 20            # Line 2 runs second
print(name)        # Line 3 runs third

But real programs need to MAKE DECISIONS and REPEAT ACTIONS. Think about it:

  • ATM machine: "IF your PIN is correct, THEN show your balance. OTHERWISE, say 'wrong PIN'"
  • Video game: "WHILE the player is alive, KEEP checking for enemy attacks"
  • Login system: "FOR each failed login attempt, wait longer before allowing another try"

The Choose Your Own Adventure Book Metaphor

Remember "Choose Your Own Adventure" books? They don't go page 1, 2, 3, 4 in order. Instead:

  • "If you open the door, turn to page 47"
  • "If you run away, turn to page 82"

Control flow is how your program decides which "page" (code) to run next. It's the difference between:

  • Sequential flow: Read every page in order (boring, no choices)
  • Control flow: Make decisions, skip sections, repeat chapters (exciting, interactive)

Python gives us THREE main control flow tools:

  • if statements - Make decisions (the bouncer at a club checking ID)
  • Loops - Repeat actions (the assembly line worker doing the same task)
  • Functions - Save recipes of code to reuse (the cookbook you reference)

Let's master each one with EXTREME detail!

Part 2: if Statements - The Bouncer

The Bouncer Metaphor

Imagine a bouncer at a nightclub entrance. Their job is simple:

  • IF your ID shows you're 21 or older → "Come in, enjoy!"
  • ELSE (otherwise) → "Sorry, you can't enter"

The bouncer is making a DECISION based on a CONDITION. Python's if statement works the EXACT same way - it checks a condition and decides what code to run.

The Basic if Statement

An if statement has this structure: 

if condition_is_true:
    # Run this code (notice the indent!)

Demo: Simple if Statement

age = 25

if age >= 21:
    print("You can enter the club!")
    print("Enjoy your night!")

print("Have a great day!")  # This ALWAYS runs (not indented)

Output:

You can enter the club!
Enjoy your night!
Have a great day!

CRITICAL: Indentation Matters!

Python uses indentation (spaces at the start of lines) to show which code belongs to the if statement. This is NOT optional - it's how Python knows what to run! 

# CORRECT (4 spaces or 1 tab indent)
if age >= 21:
    print("This runs if true")

# WRONG (no indent = error!)
if age >= 21:
print("ERROR! Python doesn't know this belongs to if")

The if-else Statement (Two Options)

What if you want to do something DIFFERENT when the condition is false? Use else:

Demo: if-else Statement

age = 18

if age >= 21:
    print("Welcome! Come on in!")
else:
    print("Sorry, you must be 21 or older.")
    print("Come back in a few years!")

print("Goodbye!")  # Always runs

Output:

Sorry, you must be 21 or older.
Come back in a few years!
Goodbye!

The if-elif-else Statement (Multiple Options)

What if you have MORE than two options? Use elif (short for "else if"):

The Ticket Booth Metaphor

Imagine a movie theater ticket booth:

  • IF age < 13 → Child ticket ($5)
  • ELIF age < 65 → Adult ticket ($12)
  • ELSE → Senior ticket ($7)

Python checks conditions in order and runs the FIRST one that's true, then STOPS checking!

Demo: if-elif-else Chain

age = 30

if age < 13:
    price = 5
    print("Child ticket: $5")
elif age < 65:
    price = 12
    print("Adult ticket: $12")
else:
    price = 7
    print("Senior ticket: $7")

print(f"Total: ${price}")

Output:

Adult ticket: $12
Total: $12

Comparison Operators (How to Check Conditions)

Here are ALL the ways to compare values in if statements: 

==  # Equal to (Is this EXACTLY the same?)
!=  # Not equal to (Is this DIFFERENT?)
>   # Greater than
<   # Less than
>=  # Greater than or equal to
<=  # Less than or equal to

# Examples:
age == 21      # Is age exactly 21?
name != "Bob"  # Is name NOT Bob?
score > 90     # Is score greater than 90?

Common Mistake: = vs ==

  • = is ASSIGNMENT (put value into variable): age = 21
  • == is COMPARISON (check if equal): age == 21
# WRONG - This CHANGES age to 21!
if age = 21:  # ERROR!

# CORRECT - This CHECKS if age is 21
if age == 21:  # Good!

Combining Conditions (and, or, not)

You can combine multiple conditions using and, or, and not:

Demo: Combining Conditions

age = 25
has_ticket = True

# AND - Both must be true
if age >= 18 and has_ticket:
    print("Welcome to the concert!")

# OR - At least one must be true
is_weekend = True
is_holiday = False

if is_weekend or is_holiday:
    print("Yay! No work today!")

# NOT - Reverses the condition
is_raining = False

if not is_raining:
    print("Let's go to the park!")

Part 3: Loops - The Assembly Line

The Assembly Line Metaphor

Imagine a factory assembly line where a worker:

  • Picks up a box
  • Puts a sticker on it
  • Puts it on the conveyor belt
  • REPEATS this 1,000 times

You wouldn't write the instructions 1,000 times! Instead: "Do this task, REPEAT until all boxes are done."

Loops are Python's way of saying "repeat this code multiple times without writing it multiple times."

The for Loop (Loop a Specific Number of Times)

Use for when you know EXACTLY how many times to repeat, or when you want to go through each item in a list:

Demo: Basic for Loop

# Print numbers 0 through 4
for i in range(5):
    print(i)

# Output:
# 0
# 1
# 2
# 3
# 4

range(5) creates numbers from 0 to 4 (5 total numbers). The variable i takes on each value one at a time.

Demo: Loop Through a List

students = ["Alice", "Bob", "Charlie"]

for student in students:
    print(f"Hello, {student}!")

# Output:
# Hello, Alice!
# Hello, Bob!
# Hello, Charlie!

Understanding range()

range(5)         # Numbers 0, 1, 2, 3, 4
range(1, 6)      # Numbers 1, 2, 3, 4, 5 (start at 1, stop before 6)
range(0, 10, 2)   # Numbers 0, 2, 4, 6, 8 (start, stop, step)

The while Loop (Loop Until a Condition Changes)

Use while when you DON'T know exactly how many times to repeat, but you know WHEN to stop:

The Video Game Metaphor

Think of a video game:

  • WHILE the player has health > 0, KEEP playing
  • When health reaches 0, STOP (game over)

You don't know HOW MANY attacks it will take to reach 0 health, but you know the condition to check: "Is health > 0?"

Demo: while Loop

countdown = 5

while countdown > 0:
    print(countdown)
    countdown -= 1  # Same as: countdown = countdown - 1

print("Blast off!")

# Output:
# 5
# 4
# 3
# 2
# 1
# Blast off!

Danger: Infinite Loops!

If your while condition NEVER becomes false, the loop runs FOREVER (infinite loop): 

countdown = 5

while countdown > 0:
    print(countdown)
    # OOPS! Forgot to decrease countdown!
    # This will print 5 forever!

ALWAYS make sure your while loop has a way to end! (Press Ctrl+C to stop infinite loops)

Breaking Out of Loops (break and continue)

Sometimes you need to exit a loop early or skip to the next iteration:

Demo: break (Exit Loop Immediately)

# Find first number divisible by 7
for num in range(1, 100):
    if num % 7 == 0:
        print(f"Found it! {num}")
        break  # Stop the loop immediately

# Output: Found it! 7

Demo: continue (Skip to Next Iteration)

# Print only odd numbers
for num in range(1, 11):
    if num % 2 == 0:  # If even
        continue  # Skip this iteration, go to next number
    print(num)

# Output: 1, 3, 5, 7, 9

Part 4: Functions - The Recipe

The Recipe Metaphor

Imagine you bake chocolate chip cookies frequently. Do you look up the recipe every time? No! You:

  1. Write down the recipe ONCE (define the function)
  2. Give it a name: "Chocolate Chip Cookies"
  3. Use it whenever needed: "Make Chocolate Chip Cookies" (call the function)

A function in Python is EXACTLY like a recipe:

  • You write code ONCE
  • Give it a name
  • Use (call) it as many times as you want
  • Can accept inputs (ingredients) and produce outputs (the finished dish)

Creating Your First Function

Use the def keyword (short for "define") to create a function:

Demo: Basic Function

# DEFINE the function (write the recipe)
def greet():
    print("Hello!")
    print("Welcome to Python!")

# CALL the function (use the recipe)
greet()  # Output: Hello! \n Welcome to Python!
greet()  # Can use it again!
greet()  # And again!

Functions with Parameters (Adding Ingredients)

Parameters are inputs your function can accept. Think of them as ingredients in a recipe:

Demo: Function with Parameters

# Define function that accepts a NAME
def greet_person(name):
    print(f"Hello, {name}!")
    print(f"Nice to meet you, {name}!")

# Call function with DIFFERENT inputs
greet_person("Alice")   # Output: Hello, Alice! ...
greet_person("Bob")     # Output: Hello, Bob! ...
greet_person("Charlie") # Output: Hello, Charlie! ...

Demo: Function with Multiple Parameters

def add_numbers(num1, num2):
    result = num1 + num2
    print(f"{num1} + {num2} = {result}")

add_numbers(5, 3)   # Output: 5 + 3 = 8
add_numbers(10, 20) # Output: 10 + 20 = 30

Functions with Return Values (Getting Results Back)

So far, our functions just PRINT things. But what if you want to GET a result back to use later? Use return:

The Vending Machine Metaphor

When you put money into a vending machine:

  • You give INPUT (coins)
  • Machine does work (processes your selection)
  • You get OUTPUT (your snack)

The return statement is the "snack" that comes out - the result you can use elsewhere in your code!

Demo: Function with Return

def add_numbers(num1, num2):
    result = num1 + num2
    return result  # Give back the result

# Now we can SAVE the result!
total = add_numbers(5, 3)
print(f"The total is: {total}")  # Output: The total is: 8

# Can use it in calculations!
double_total = add_numbers(5, 3) * 2
print(double_total)  # Output: 16

print() vs return

  • print() - Shows text to the user (one-way communication)
  • return - Sends value back to your code to use later (two-way communication)

Part 5: Complete Program - Password Strength Checker

Let's combine EVERYTHING we learned into one complete program:

Complete Working Program

def check_password_strength(password):
    """Check if a password is strong enough."""
    
    # Check length
    if len(password) < 8:
        return "Weak - Too short! Use at least 8 characters."
    
    # Check for numbers
    has_number = False
    for char in password:
        if char.isdigit():
            has_number = True
            break
    
    if not has_number:
        return "Medium - Add some numbers!"
    
    # Check for uppercase letters
    has_upper = False
    for char in password:
        if char.isupper():
            has_upper = True
            break
    
    if not has_upper:
        return "Medium - Add some uppercase letters!"
    
    return "Strong - Great password!"

# Main program
print("=== Password Strength Checker ===")
print()

# Test different passwords
passwords = ["abc", "password123", "SecurePass123"]

for pwd in passwords:
    strength = check_password_strength(pwd)
    print(f"Password: '{pwd}'")
    print(f"Strength: {strength}")
    print()

# Let user test their own password
user_password = input("Enter your password to test: ")
result = check_password_strength(user_password)
print(f"\nYour password is: {result}")

How to run this:

  1. Save as password_checker.py
  2. Run: python password_checker.py

Sample Output:

=== Password Strength Checker ===

Password: 'abc'
Strength: Weak - Too short! Use at least 8 characters.

Password: 'password123'
Strength: Medium - Add some uppercase letters!

Password: 'SecurePass123'
Strength: Strong - Great password!

Enter your password to test:

This program uses:

  • Functions - check_password_strength() is reusable
  • if-elif-else - Makes decisions about password strength
  • for loops - Checks each character in password
  • break - Stops checking once we find what we need
  • return - Sends result back to caller
