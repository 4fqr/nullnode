üéØ What is OWASP Juice Shop?

OWASP Juice Shop is a deliberately insecure web application built for learning about web security. It's like a playground where you can practice hacking legally and safely - it's an online juice shop with hundreds of security vulnerabilities waiting to be exploited!

üí° What Does "OWASP" Mean?

OWASP (Open Web Application Security Project) is a non-profit organization focused on improving software security. They publish the OWASP Top 10 - a list of the most critical web application security risks. Juice Shop covers ALL of them!

What You'll Learn

  • SQL Injection - Manipulate database queries to access unauthorized data
  • Cross-Site Scripting (XSS) - Inject malicious JavaScript into web pages
  • Authentication Bypass - Break login systems and access other accounts
  • Broken Access Control - Access resources you shouldn't be able to
  • Security Misconfiguration - Find exposed admin panels and sensitive files
  • Cross-Site Request Forgery (CSRF) - Make users perform unwanted actions
  • API Exploitation - Abuse backend APIs to manipulate data
  • Cryptographic Failures - Crack weak passwords and encryption

üíâ Challenge 1: SQL Injection Login Bypass

HIGH SEVERITY

What is SQL Injection?

üí° SQL Injection Explained Simply

Imagine a library where you ask for a book by filling out a form. The librarian takes your request and runs: SELECT * FROM books WHERE title = 'YOUR_INPUT'

But what if instead of a book title, you write: nothing' OR '1'='1?

The query becomes: SELECT * FROM books WHERE title = 'nothing' OR '1'='1'

Since '1'='1' is ALWAYS TRUE, this returns ALL books! That's SQL injection - injecting SQL code into user input to change query logic.

The Vulnerability

Juice Shop's login page doesn't properly escape user input. When you type an email and password, the server runs something like: 

SELECT * FROM users WHERE email = 'YOUR_EMAIL' AND password = 'YOUR_PASSWORD'

If we inject SQL code into the email field, we can bypass the password check entirely!

The Exploit

  1. Go to the Login Page
    Click "Account" ‚Üí "Login" in the top right
  2. Inject SQL into Email Field
    Email: ' OR 1=1--
    Password: anything (doesn't matter)
  3. Click Login
    You're now logged in as the first user in the database (usually admin)!

Why It Works

The query becomes:

SELECT * FROM users WHERE email = '' OR 1=1--' AND password = 'anything'

Let's break this down:

  • '' - Empty email (false)
  • OR - "Or" operator
  • 1=1 - Always true!
  • -- - SQL comment symbol (ignores everything after, including the password check)
üî• Real-World Impact

SQL injection has been used to steal millions of credit cards, expose entire user databases, and deface websites. It's been #1 on the OWASP Top 10 multiple times. NEVER trust user input!

Other SQL Injection Payloads to Try

# Login as admin specifically
admin'--
admin@juice-sh.op'--

# Bypass without knowing any username
' OR '1'='1'--
' OR 'a'='a'--

# Union-based injection (extract data)
' UNION SELECT NULL,NULL,NULL--

üö® Challenge 2: Cross-Site Scripting (XSS)

HIGH SEVERITY

What is XSS?

üí° XSS Explained Simply

Cross-Site Scripting (XSS) is when you inject JavaScript code into a website, and that code runs in other users' browsers. Imagine leaving a sticky note on a public bulletin board, but the note is actually a command that makes people's phones ring when they look at it!

Example: If a website displays user comments without sanitizing them, you could post a comment like <script>alert('HACKED!')</script> and everyone who views that page would see your alert!

Finding the XSS Vulnerability

Juice Shop has several XSS vulnerabilities. Let's exploit the search function!

  1. Navigate to the Search Bar
    Look at the top of the Juice Shop homepage
  2. Test for XSS
    Type: <script>alert('XSS')</script>
    Press Enter
  3. Watch the Alert Pop Up
    If you see an alert box saying "XSS", congratulations - you found an XSS vulnerability!

More XSS Payloads to Try

# Basic alert
<script>alert('XSS')</script>

# Image tag XSS (when <script> is blocked)
<img src=x onerror="alert('XSS')">

# Steal cookies (malicious!)
<script>alert(document.cookie)</script>

# Redirect users
<script>window.location='http://evil.com'</script>

# DOM-based XSS (via URL)
http://localhost:3000/#/search?q=<script>alert('XSS')</script>

Types of XSS

Reflected XSS

Your payload is immediately reflected back in the response. Example: Search results showing your search term without sanitization.

Stored XSS

Your payload is saved in the database and affects everyone who views it. Example: Malicious comment that executes when others read comments.

DOM XSS

The vulnerability is in client-side JavaScript code that processes user input. Example: JavaScript reading URL parameters and inserting them into the page.

üî• Real-World Impact

XSS can steal session cookies (hijacking accounts), redirect users to phishing sites, deface websites, or install malware. Major sites like Facebook, Twitter, and eBay have all had XSS vulnerabilities. It's EXTREMELY common!

  • Three types of XSS: Reflected, Stored, DOM
  • How to test for XSS with payloads
  • Alternative XSS vectors when <script> is blocked
  • Why input sanitization is critical

üîì Challenge 3: Password Reset Bypass

HIGH SEVERITY

What is Broken Authentication?

üí° Broken Authentication Explained

Authentication is how websites verify you are who you claim to be (usually username + password). Broken authentication means there are flaws in this system that let attackers bypass it - like having a lock on your door but the key is taped to the door frame!

The Vulnerability

Juice Shop has a password reset function that asks security questions. But if you can guess or find out someone's security answer, you can reset their password!

  1. Go to Forgot Password
    Click "Account" ‚Üí "Login" ‚Üí "Forgot your password?"
  2. Try Admin's Email
    Enter: admin@juice-sh.op
  3. Find Their Security Question
    The question is: "Your eldest siblings middle name?"
  4. Research the Answer
    üí° OSINT (Open Source Intelligence)

    Look around Juice Shop! Check the "About Us" page. You'll find information about the company founder's family. This is called OSINT - gathering information from publicly available sources.

  5. Answer the Question
    The answer is hidden in Juice Shop's lore (hint: check the photo gallery!)
  6. Reset the Password
    Set a new password and login as admin!

Other Authentication Flaws in Juice Shop

  • Weak Passwords - Users with passwords like "admin123" can be brute-forced
  • JWT Manipulation - JSON Web Tokens can sometimes be tampered with (check cookies in DevTools!)
  • Session Fixation - Reusing session tokens across different users

üì° Challenge 4: Hidden API Exploitation

MEDIUM SEVERITY

What is an API?

üí° API Explained Simply

API (Application Programming Interface) is how the frontend (what you see) talks to the backend (server/database). When you click "Add to Cart", your browser sends an API request like POST /api/cart {productId: 5} to the server.

APIs often have hidden endpoints that aren't linked in the UI but are still accessible if you know the URL!

Finding Hidden APIs

  1. Open Browser DevTools
    Press F12 ‚Üí Go to "Network" tab
  2. Interact with Juice Shop
    Browse products, add items to cart, search for products
  3. Watch Network Requests
    You'll see requests like: 
    /api/Products
/api/BasketItems
/api/Users
/api/SecurityQuestions
  4. Test Endpoints Manually
    Try visiting: http://localhost:3000/api/Users
    You might see ALL users in the database!

Exploiting APIs with Python

#!/usr/bin/python3
# Extract all users from API

import requests

# Target API endpoint
url = "http://localhost:3000/api/Users"

# Send GET request
response = requests.get(url)

# Parse JSON response
users = response.json()

# Display all emails
print("Found users:")
for user in users['data']:
    print(f"- {user['email']}")

Common API Vulnerabilities

No Authentication

APIs that don't check if you're logged in. Anyone can access sensitive data!

Broken Authorization

You can access other users' data by changing IDs: /api/user/1 ‚Üí /api/user/2

Mass Assignment

Modify fields you shouldn't by adding them to requests: {"isAdmin": true}

Rate Limiting Missing

APIs with no rate limits can be brute-forced or DDoS'd with automated requests
