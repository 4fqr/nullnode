Chapter 02: Data Structures

Your Containers for Storing Multiple Pieces of Information

Part 1: What ARE Data Structures?

Remember in Chapter 1 when we learned about variables? We discovered that a variable is like a labeled box that holds ONE piece of information: 

name = "Sarah"  # One box holding one name
age = 25       # One box holding one number

But what if you're building a school program and need to store 1,000 student names? Are you going to make 1,000 separate variables? 

student1 = "Alice"
student2 = "Bob"
student3 = "Charlie"
# ... 997 more variables? NIGHTMARE!

This is where data structures come in. They're like CONTAINERS that can hold MULTIPLE pieces of information in an organized way.

The Storage Room Metaphor

Think of your programming needs like organizing a storage room:

  • Single variables = Individual boxes scattered on the floor (hard to manage many)
  • Data structures = Shelving units, filing cabinets, and organizers (easy to manage thousands of items)

You wouldn't store 1,000 books by placing each one randomly on the floor. You'd use a BOOKSHELF. Data structures are your "bookshelves" for organizing information in code.

Python gives us FOUR main data structures. Think of them as four different types of storage containers, each perfect for different situations:

  • Lists - An apartment building (numbered floors you can visit, modify, rearrange)
  • Dictionaries - A phone book (look things up by name, not by position)
  • Tuples - A sealed package (once packed, contents can't change)
  • Sets - A VIP club (no duplicates allowed, everyone is unique)

Each type is a tool designed for specific jobs. Let's explore them ONE AT A TIME with EXTREME detail.

Part 2: Lists - The Apartment Building

The Apartment Building Metaphor

Imagine an apartment building with numbered floors:

  • Floor 0 (ground floor) - First resident
  • Floor 1 - Second resident
  • Floor 2 - Third resident
  • And so on...

A Python list is EXACTLY like this building:

  • Each "floor" is an index (position number)
  • Each floor can hold ONE piece of data (the "resident")
  • You can visit any floor directly by its number
  • You can kick residents out, move new ones in, rearrange everything
  • IMPORTANT: Python buildings start counting at Floor 0, not Floor 1!

Creating Your First List

Lists use square brackets [ ] and separate items with commas:

Demo: Creating Lists

# A list of student names
students = ["Alice", "Bob", "Charlie", "Diana"]

# A list of test scores
scores = [95, 87, 92, 88]

# A list mixing different types (perfectly fine!)
profile = ["Sarah", 25, "Engineer", True]

# An empty list (like an empty apartment building)
empty_list = []

print(students)  # Output: ['Alice', 'Bob', 'Charlie', 'Diana']

Accessing List Items (Visiting Floors)

To access an item in a list, use its index (position number) in square brackets. Remember: counting starts at 0!

Demo: Accessing List Items

students = ["Alice", "Bob", "Charlie", "Diana"]
# Index:       0         1          2           3

print(students[0])  # Output: Alice (first floor/index 0)
print(students[2])  # Output: Charlie (third floor/index 2)

# You can also count BACKWARDS from the end!
print(students[-1])  # Output: Diana (last item)
print(students[-2])  # Output: Charlie (second from last)

Common Mistake: Index Out of Range

If you try to access an index that doesn't exist (like Floor 10 in a 4-story building), Python will crash with an error: 

students = ["Alice", "Bob"]  # Only 2 items (indexes 0 and 1)
print(students[5])  # ERROR! No index 5 exists!

Always check how many items your list has using len(): 

print(len(students))  # Output: 2 (so valid indexes are 0 and 1)

Modifying Lists (Renovating the Apartment)

Lists are MUTABLE (changeable). You can replace items, add new ones, or remove them.

Demo: Changing List Items

students = ["Alice", "Bob", "Charlie"]

# Replace an item (kick Bob out, move Eve in)
students[1] = "Eve"
print(students)  # Output: ['Alice', 'Eve', 'Charlie']

# Add item to the END of list
students.append("Diana")
print(students)  # Output: ['Alice', 'Eve', 'Charlie', 'Diana']

# Insert item at SPECIFIC position
students.insert(1, "Frank")  # Insert Frank at index 1
print(students)  # Output: ['Alice', 'Frank', 'Eve', 'Charlie', 'Diana']

# Remove item by VALUE
students.remove("Eve")
print(students)  # Output: ['Alice', 'Frank', 'Charlie', 'Diana']

# Remove item by INDEX and get it back
removed_student = students.pop(2)  # Remove index 2
print(removed_student)  # Output: Charlie
print(students)  # Output: ['Alice', 'Frank', 'Diana']

List Slicing (Visiting Multiple Floors)

You can grab a "slice" (range) of items from a list using list[start:end]:

Demo: List Slicing

numbers = [10, 20, 30, 40, 50, 60]
# Index:    0    1    2    3    4    5

# Get items from index 1 to 4 (doesn't include 4)
print(numbers[1:4])  # Output: [20, 30, 40]

# Get first 3 items (start to index 3)
print(numbers[:3])  # Output: [10, 20, 30]

# Get last 3 items (index 3 to end)
print(numbers[3:])  # Output: [40, 50, 60]

# Copy entire list
copy = numbers[:]
print(copy)  # Output: [10, 20, 30, 40, 50, 60]

Part 3: Dictionaries - The Phone Book

The Phone Book Metaphor

Lists work great when you want to access items by POSITION (index). But what if you want to look things up by NAME instead of by number?

Think about a phone book:

  • You DON'T look up "Who's the 347th person in this book?"
  • You DO look up "What's Alice's phone number?"

A Python dictionary works like a phone book:

  • Each entry has a KEY (the person's name)
  • Each key has a VALUE (their phone number)
  • You look things up by the KEY, not by position
  • Keys must be UNIQUE (can't have two "Alice" entries)

Creating Your First Dictionary

Dictionaries use curly braces { } and have KEY: VALUE pairs separated by commas:

Demo: Creating Dictionaries

# A dictionary of phone numbers
phone_book = {
    "Alice": "555-1234",
    "Bob": "555-5678",
    "Charlie": "555-9012"
}

# A dictionary for a student profile
student = {
    "name": "Sarah",
    "age": 20,
    "major": "Computer Science",
    "gpa": 3.8
}

# An empty dictionary
empty_dict = {}

print(student)  # Output: {'name': 'Sarah', 'age': 20, ...}

Accessing Dictionary Values (Looking Up in Phone Book)

To get a value, use its KEY in square brackets:

Demo: Accessing Dictionary Values

student = {
    "name": "Sarah",
    "age": 20,
    "major": "Computer Science"
}

print(student["name"])   # Output: Sarah
print(student["age"])    # Output: 20

# Safer way using .get() (returns None if key doesn't exist)
print(student.get("gpa"))        # Output: None (key doesn't exist)
print(student.get("gpa", 0.0))  # Output: 0.0 (default value)

Modifying Dictionaries (Updating the Phone Book)

Demo: Modifying Dictionaries

student = {"name": "Sarah", "age": 20}

# Change existing value
student["age"] = 21
print(student)  # Output: {'name': 'Sarah', 'age': 21}

# Add new key-value pair
student["major"] = "Engineering"
print(student)  # Output: {'name': 'Sarah', 'age': 21, 'major': 'Engineering'}

# Remove key-value pair
del student["age"]
print(student)  # Output: {'name': 'Sarah', 'major': 'Engineering'}

# Get all keys
print(student.keys())    # Output: dict_keys(['name', 'major'])

# Get all values
print(student.values())  # Output: dict_values(['Sarah', 'Engineering'])

Part 4: Tuples - The Sealed Package

The Sealed Package Metaphor

Imagine you're mailing a package. Once you seal it with tape, you CAN'T open it and rearrange what's inside. You can still LOOK at the label and know what's in there, but you can't MODIFY the contents without destroying the package.

A Python tuple is like this sealed package:

  • It holds multiple items (like a list)
  • You can ACCESS items by index (like a list)
  • But you CANNOT change, add, or remove items after creation
  • They're IMMUTABLE (unchangeable)

Why use tuples? When you want to protect data from accidental modification. For example, GPS coordinates shouldn't change after you set them!

Creating Tuples

Tuples use parentheses ( ) instead of square brackets:

Demo: Creating and Using Tuples

# GPS coordinates (latitude, longitude) - shouldn't change!
location = (40.7128, -74.0060)

# RGB color values (red, green, blue)
color = (255, 0, 128)

# Personal info you don't want changed
person = ("Alice", 25, "Engineer")

# Access items by index (just like lists)
print(location[0])  # Output: 40.7128
print(color[1])     # Output: 0

# Try to modify (this will CRASH!)
location[0] = 50.0  # ERROR! Tuples can't be changed!

When to Use Tuples vs Lists

  • Use LISTS when data might change (student names, shopping cart items)
  • Use TUPLES when data should NEVER change (birthdate, coordinates, IDs)

Part 5: Sets - The VIP Club

The VIP Club Metaphor

Imagine a VIP club with strict rules:

  • No duplicates allowed - Alice can't enter twice, even if she tries
  • No guaranteed order - People might be at the bar, dance floor, or lounge (not standing in a line)
  • Membership checking is SUPER FAST - Bouncer has a list and can instantly say "yes you're in" or "no you're not"

A Python set is EXACTLY like this club:

  • Only unique items (automatically removes duplicates)
  • No indexes (items aren't in a specific order)
  • Lightning-fast membership checking

Creating Sets

Sets use curly braces { } like dictionaries, but without KEY: VALUE pairs:

Demo: Creating and Using Sets

# Create a set of unique numbers
numbers = {1, 2, 3, 4, 5}
print(numbers)  # Output: {1, 2, 3, 4, 5}

# Create set with duplicates (they'll be removed automatically!)
numbers = {1, 2, 2, 3, 3, 3, 4}
print(numbers)  # Output: {1, 2, 3, 4} (duplicates removed!)

# Convert list to set (removes duplicates)
my_list = [1, 2, 2, 3, 3, 3]
unique_numbers = set(my_list)
print(unique_numbers)  # Output: {1, 2, 3}

# Check membership (SUPER FAST)
vip_members = {"Alice", "Bob", "Charlie"}
print("Alice" in vip_members)   # Output: True
print("Diana" in vip_members)   # Output: False

# Add member
vip_members.add("Diana")
print(vip_members)  # Output: {'Alice', 'Bob', 'Charlie', 'Diana'}

# Remove member
vip_members.remove("Bob")
print(vip_members)  # Output: {'Alice', 'Charlie', 'Diana'}

Part 6: Real Programs Using Data Structures

Program 1: Student Grade Tracker (Lists)

Complete Working Program

# Store student grades
grades = []

# Add grades
grades.append(95)
grades.append(87)
grades.append(92)

# Calculate average
average = sum(grades) / len(grades)

# Find highest and lowest
highest = max(grades)
lowest = min(grades)

print(f"Grades: {grades}")
print(f"Average: {average:.1f}")
print(f"Highest: {highest}")
print(f"Lowest: {lowest}")

How to run this:

  1. Save as grade_tracker.py
  2. Run: python grade_tracker.py

Output:

Grades: [95, 87, 92]
Average: 91.3
Highest: 95
Lowest: 87

Program 2: Inventory System (Dictionary)

Complete Working Program

# Store inventory with product names and quantities
inventory = {
    "apples": 50,
    "bananas": 30,
    "oranges": 20
}

# Sell 10 apples
inventory["apples"] -= 10

# Add new product
inventory["grapes"] = 25

# Check if product exists
if "bananas" in inventory:
    print(f"We have {inventory['bananas']} bananas")

# Show all inventory
print("\nCurrent Inventory:")
for product, quantity in inventory.items():
    print(f"  {product}: {quantity}")

Output:

We have 30 bananas

Current Inventory:
  apples: 40
  bananas: 30
  oranges: 20
  grapes: 25
